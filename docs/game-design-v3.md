# Manas ゲーム設計 v3 — 統合確定仕様

> **目的**: 3〜18歳の神経発達症児（ASD・ADHD・ID）が、ゲームプレイを通じて認知機能の近接転移的改善を達成し、同時に認知プロファイルを継続測定できるアプリの設計仕様
>
> **根拠文献**: Miyake et al. (2000), Diamond (2013), Wiebe et al. (2008), Verbruggen et al. (2019), Tamm et al. (2012), Kollins et al. (2020, STARS-ADHD), Melby-Lervåg & Hulme (2013), Valeri et al. (2023, Neuropsychology Review) 他
>
> **設計原則**: far transferの過度な主張を避け、「近接スキル改善」と「特性理解・得意の発見」を価値の中心に据える。Lumosity FTC罰金（$2M, 2016年）の教訓を踏まえ、「IQが上がる」「学力が向上する」等のFar Transfer主張は一切行わない
>
> **エンゲージメント原則**: 科学的に有効なゲーム設計であっても、子どもが継続プレイしなければ効果はゼロ。Duolingo等の行動設計知見を障がい児向けに適応し、「測定精度」と「継続率」の両立を図る
>
> **競合ポジショニング**: 3障害（ASD・ADHD・ID）を同一プラットフォームで障害別DDA最適化する設計は、EndeavorRx・CogniFit・Cogmed・ACTIVATE・BrainHQの全5社に前例がない。社会認知・情動調整ドメインのカバレッジ、3歳からの対応年齢も独自の優位性
>
> **v2からの主要変更点**: ID重度DDAターゲット引き上げ（85→90%）、オンボーディング抜本再設計（25画面→3画面＋後置）、感覚過敏コントロールUI新設、移行期予告UI新設、保護者ダッシュボード拡充、ストリーク保護バッファ追加、試行間インターバルの障害別設定、エビデンスロードマップ追加

## 1. ドメイン構成: 15 → 5カテゴリ

幼児期（6歳未満）のEFは統計的に未分化（Wiebe et al., 2008）。15ドメインの独立測定は測定ノイズを増やし、保護者を混乱させる。

### 5カテゴリ

| # | カテゴリ | 内容 | 対応ゲーム |
|---|--------|------|----------|
| 1 | **注意・抑制** | 持続的注意、選択的注意、反応抑制 | ひかりレスキュー |
| 2 | **記憶・学習** | 視空間WM、聴覚WM、遅延記憶 | おぼえてすすむ |
| 3 | **柔軟性・実行制御** | ルール切替、計画、問題解決 | ルールチェンジ、たんけんめいろ |
| 4 | **知覚・空間・推論** | 視空間認知、パターン認識、微細運動 | くるくるパズル、タッチアドベンチャー |
| 5 | **社会認知・言語** | 表情理解、受容語彙 | きもちフレンズ、ことばとえほん |

**表示**: 保護者向けレポートはこの5カテゴリで表示。内部データは複数指標を保持。

## 3. 各ゲーム詳細仕様

### Game 1: ひかりレスキュー（注意・抑制）

**世界観**: 光る生き物が暗闇で迷子になっている。タッチして助ける。トゲ付き（赤い光）は触らない。

**ベース検査**: CPT (K-CPT準拠) + Go/No-Go

**レベル構成**:

| レベル帯 | 内容 | 対象 |
|---------|------|------|
| Lv 1-3 | Go only。光る生き物をタッチするだけ。No-Go刺激なし | Tier 1（MA < 2歳） |
| Lv 4-8 | Go/No-Go。標的1種 vs 非標的1種。絵柄弁別 | Tier 2（MA 2-4歳） |
| Lv 9-15 | CPT形式。ISI変動、標的/非標的比率変動 | Tier 2上位 |
| Lv 16-20 | Stop-signal追加。Go中に停止信号が出る | Tier 3（MA > 4歳） |
| Lv 21-25 | 複数標的、長時間持続、信号遅延変動 | Tier 3上位 |

**DDAパラメータ（調整優先順）**:
1. 刺激弁別性（標的と非標的の類似度）
2. 標的/非標的比率
3. ISI（刺激間間隔）
4. ブロック長（3歳: 12試行 → 8歳: 30試行）
5. 時間制限（最後に導入）

**主要測定指標**:
- d'（信号検出感度）
- RT variability（CV = SD/Mean）← 注意欠如の最も敏感な指標 (Tamm et al., 2012)
- False alarm率
- Omission率
- Stop-signal RT（Lv16以上のみ）

**注意事項**:
- 3歳児にSSRTは不安定 (Verbruggen et al., 2019)。Lv16未満では算出しない
- K-CPT発想で刺激は pictures-only、短ブロック
- Go優勢文脈で抑制を測る（No-Go比率は最大30%）

---

### Game 2: おぼえてすすむ（記憶・学習）

**世界観**: ロボットが色/場所/音のパターンを示す。真似して入力する。

**ベース検査**: Corsi Block + DMS (Delayed Matching to Sample)

**レベル構成**:

| レベル帯 | 内容 | 対象 |
|---------|------|------|
| Lv 1-3 | 順方向2要素。遅延なし | Tier 1-2（MA < 3歳） |
| Lv 4-8 | 順方向3-4要素。場所+色 | Tier 2（MA 3-4歳） |
| Lv 9-13 | 順方向5要素 + DMS（遅延0-4秒） | Tier 2-3 |
| Lv 14-18 | 逆方向導入（3要素から） | Tier 3（MA > 5歳） |
| Lv 19-25 | 逆方向4-5要素 + DMS遅延8秒 | Tier 3上位 |

**DDAパラメータ（調整優先順）**:
1. 系列長（span）
2. 刺激モダリティ（視覚のみ → 視覚+聴覚）
3. 遅延時間（DMS: 0 → 2 → 4 → 8秒。12秒は幼児には長すぎる）
4. 干渉刺激の有無

**主要測定指標**:
- Forward span
- Backward span（Lv14以上）
- DMS正答率（遅延別）
- 衝動性指標（遅延中の早押し回数）

**注意事項**:
- backward spanはMA 5歳未満には不可。順方向span ≥ 3を前提条件とする
- 遅延12秒は採用しない（幼児のWM保持限界を超える）

---

### Game 3: ルールチェンジ（柔軟性・実行制御）

**世界観**: 魔法のカード遊び。ルールが変わる（色で分ける → 形で分ける）。

**ベース検査**: DCCS (Dimensional Change Card Sort)

**レベル構成**:

| レベル帯 | 内容 | 対象 |
|---------|------|------|
| Lv 1-3 | Pre-switchのみ（1ルールで分類） | Tier 1-2（MA < 3歳） |
| Lv 4-8 | Pre-switch → Post-switch（ルール切替1回） | Tier 2（MA 3-4歳） |
| Lv 9-13 | 複数切替。切替頻度が上がる | Tier 2-3 |
| Lv 14-18 | Border条件（枠の有無でルール決定） | Tier 3（MA > 5歳） |
| Lv 19-25 | 3次元以上、逆転ルール | Tier 3上位 |

**DDAパラメータ（調整優先順）**:
1. ルール数（1 → 2 → 3）
2. 切替頻度
3. 次元数
4. 応答時間制限（最後に導入）

**主要測定指標**:
- Pre-switch正答率
- Post-switch正答率
- 保続エラー率（切替後に旧ルールで回答した率）
- 切替コスト（切替直後のRT - 定常RT）

**注意事項**:
- DCCSは3歳で保続、5歳で改善という明確な発達曲線がある (Zelazo, 2006)
- Border条件はMA 5歳未満には混乱の元。ロック対象
- **言語指示ではなくデモ中心**で設計（後述の指示L1-L2参照）

---

### Game 4: くるくるパズル（知覚・空間・推論）

**世界観**: 壊れたおもちゃを直す。パーツを回転・はめ込み・補完する。

**ベース検査**: Mental Rotation (2D) + 系列補完 (Raven CPM簡略版)

**レベル構成**:

| レベル帯 | 内容 | 対象 |
|---------|------|------|
| Lv 1-3 | 形合わせ（回転なし、同一形状マッチ） | Tier 1-2 |
| Lv 4-8 | 2D回転（90°単位）、具体的な絵柄 | Tier 2 |
| Lv 9-13 | 2D回転（自由角度）+ 鏡像弁別導入 | Tier 2-3 |
| Lv 14-18 | 具体的絵柄の系列補完（「次は何？」） | Tier 3 |
| Lv 19-25 | 抽象図形の系列・マトリクス補完 | Tier 3上位 |

**DDAパラメータ（調整優先順）**:
1. 回転角度（0° → 90° → 自由）
2. 刺激の具体性（具体物 → 抽象図形）
3. 選択肢数（2 → 4）
4. 鏡像の有無
5. 系列長

**主要測定指標**:
- 回転精度
- 系列補完正答率
- 試行時間
- 鏡像エラー率

**注意事項**:
- 3-4歳はmental rotationではなく具体的形合わせ (Hoyek et al., 2009)
- 鏡像混入は別の認知負荷を増やすため、Lv9以降に限定
- Raven形式のマトリクスはLv19以降（MA 5歳以上）に限定。それ以前は具体物の系列補完

---

### Game 5: たんけんめいろ（計画・問題解決）

**世界観**: 宝物を求めて迷路を探検。上位帯では鍵→扉の順序計画が必要。

**ベース検査**: Maze + ToL要素（上位帯のみ）

**レベル構成**:

| レベル帯 | 内容 | 対象 |
|---------|------|------|
| Lv 1-3 | 一本道。スワイプで進む。行き止まりなし | Tier 1 |
| Lv 4-8 | 分岐1-2箇所。行き止まりあり | Tier 2 |
| Lv 9-13 | 分岐3-5箇所。アイテム収集 | Tier 2-3 |
| Lv 14-18 | 鍵→扉の順序性（ToL的計画要素） | Tier 3 |
| Lv 19-25 | 複数鍵・複数扉の順序最適化 | Tier 3上位 |

**DDAパラメータ（調整優先順）**:
1. 迷路サイズ（3×3 → 7×7+）
2. 分岐数
3. アイテム/順序制約の有無
4. 視野制限（上位帯：フォグ・オブ・ウォー）

**主要測定指標**:
- 経路効率（最短経路比）
- 戻り回数
- 停止時間（先読み指標）
- 試行錯誤パス率
- 順序エラー（鍵なしで扉に行った回数、Lv14以上）

**注意事項**:
- 迷路は直感的で、運動計画と先読みの良い指標
- ToL要素は1手課題では計画測定として意味が薄い (Krikorian 1994)。鍵→扉の順序性で代替
- エラー数だけでなく、行動パターン（戻り、停止、試行錯誤）をすべてログ化

---

### Game 6: ことばとえほん（言語）

**世界観**: 絵本の世界。読み聞かせスタイルで「○○はどれ？」

**ベース検査**: PPVT (Peabody Picture Vocabulary Test) 形式

**レベル構成**:

| レベル帯 | 内容 | 対象 |
|---------|------|------|
| Lv 1-3 | 基本名詞（りんご、いぬ等）、2択 | Tier 2（音声理解あり） |
| Lv 4-8 | 基本名詞・動詞、3択 | Tier 2 |
| Lv 9-13 | 形容詞・前置詞、4択 | Tier 2-3 |
| Lv 14-18 | 抽象語彙、文レベル理解 | Tier 3 |
| Lv 19-25 | カテゴリ分類、上位概念 | Tier 3上位 |

**DDAパラメータ（調整優先順）**:
1. 語彙難度（頻度順）
2. 選択肢数（2 → 4）
3. 選択肢の類似度（弁別性）
4. 品詞カテゴリ

**主要測定指標**:
- 受容語彙レベル（正答上限）
- 反応潜時
- カテゴリ別正答率（名詞/動詞/形容詞/抽象語）

**注意事項**:
- **TTSではなく録音音声を強く推奨**（温かみ、発音の明瞭さ）
- PPVTは2歳6ヶ月〜の適用レンジ
- 重度ID・音声理解困難な子にはロック（音声理解が本体のため）
- トレーニングとしては語彙曝露学習であり、非言語認知訓練とは別建て

---

### Game 7: きもちフレンズ（社会認知）

**世界観**: 動物キャラクターの気持ちを当てる。

**ベース検査**: 表情認識課題 + 情動Go/No-Go（上位帯）

**レベル構成**:

| レベル帯 | 内容 | 対象 |
|---------|------|------|
| Lv 1-3 | 2表情（うれしい/かなしい）マッチング | Tier 2 |
| Lv 4-8 | 4基本表情、同じ気持ちマッチ形式 | Tier 2-3 |
| Lv 9-13 | 4択ラベル選択、状況文脈あり | Tier 3 |
| Lv 14-18 | 微妙な表情差異、混合感情 | Tier 3 |
| Lv 19-25 | 情動Go/No-Go（笑顔でGo、怒りでNo-Go等） | Tier 3上位 |

**DDAパラメータ（調整優先順）**:
1. 表情数（2 → 4 → 6）
2. 回答形式（マッチ → ラベル選択）
3. 文脈の有無
4. 表情の微妙さ
5. 情動Go/No-Go条件（最上位のみ）

**主要測定指標**:
- 表情識別正答率（表情別）
- 反応潜時
- 混同パターン（どの表情をどの表情と間違えるか）
- 情動Go/No-Go成績（Lv19以上）

**注意事項**:
- 3-5歳の4択ラベルは認知負荷が高い。低年齢では「同じ気持ちマッチ」形式
- 表情訓練の般化は限定的（メタ分析）。過度な効果主張を避ける
- 静止画より動画刺激が望ましい (Baron-Cohen et al., 2001) — Phase 2で導入検討

---

### Game 8: タッチアドベンチャー（微細運動）

**世界観**: 画面上の冒険コース。タップ、スワイプ、ドラッグで進む。

**ベース検査**: Fitts' Law タスク + 微細運動評価

**レベル構成**:

| レベル帯 | 内容 | 対象 |
|---------|------|------|
| Lv 1-3 | 大ターゲット（100px+）タップ、端部回避 | Tier 1 |
| Lv 4-8 | 中ターゲット、短距離ドラッグ | Tier 2 |
| Lv 9-13 | 小ターゲット、長距離スワイプ、軌跡追従 | Tier 2-3 |
| Lv 14-18 | 精密ドラッグ、曲線追従 | Tier 3 |
| Lv 19-25 | 速度+精度の同時要求 | Tier 3上位 |

**DDAパラメータ（調整優先順）**:
1. ターゲットサイズ（大 → 小）
2. ターゲット距離（近 → 遠）
3. 操作種別（タップ → ドラッグ → 精密追従）
4. 速度要求（最後に導入）

**主要測定指標**:
- タッチ精度（ターゲット中心からの距離）
- スループット（Fitts' Index of Performance）
- パス効率（ドラッグ/スワイプの軌跡効率）
- 疲労トレンド（セッション後半の精度低下率）

**注意事項**:
- 子どものタッチ特性は成人と異なる (Fitts則が条件依存)
- 3-5歳は100px以上のターゲット、端部回避必須
- DCD群ではRT重みを下げ、精度重視で評価
- 他全ゲームのタッチログからもバックグラウンドで運動指標を算出

## 4. アダプティブ・ゲート設計

### 設計原則

- 「全員に全ゲーム」は不可能。DDAだけでは課題の構成概念が崩壊するゲームがある
- 解放条件は**年齢のみ不可**。**年齢 × オンボーディング回答 × 初回プレイ実績**の3軸
- 低閾値ゲームは全員解放、高閾値ゲームは前提条件付き解放

### ティア制

| ティア | 対象像（目安） | 解放ゲーム | 判定方法 |
|--------|--------------|-----------|---------|
| **Tier 1: 感覚・運動** | MA < 2歳、重度ID、自閉傾向強 | ひかりレスキュー(Lv1-3: Goのみ)、タッチアドベンチャー(Lv1-3)、たんけんめいろ(Lv1-3: 一本道) | デフォルト解放。ルール理解不要、単純因果のみ |
| **Tier 2: ルール・弁別** | MA 2-4歳、中度ID、幼児 | 上記 + ひかりレスキュー(Go/No-Go)、おぼえてすすむ(順方向)、ルールチェンジ(pre-switch)、くるくるパズル(形合わせ)、ことばとえほん(※音声理解あり)、きもちフレンズ(2表情マッチ) | Tier 1で正答率 > 80%、または保護者回答「2語文以上理解」 |
| **Tier 3: 実行機能** | MA > 4歳、軽度ID、定型、ASD/ADHD | 全ゲーム全レベル解放 | Tier 2での一貫した成功 + 抑制スコア基準値以上 |

### ゲーム別ゲート詳細

| ゲーム | 全員解放部分 | ロック部分 | 解放条件 |
|--------|------------|----------|---------|
| ひかりレスキュー | Lv1-8 (Go/No-Go) | Lv16+ (Stop-signal) | 抑制プロファイル十分 + MA ≥ 4 |
| おぼえてすすむ | Lv1-8 (順方向) | Lv14+ (逆方向) | 順方向span ≥ 3 + MA ≥ 5 |
| ルールチェンジ | Lv1-8 (pre/post-switch) | Lv14+ (border条件) | 2ルール理解成功 + MA ≥ 5 |
| くるくるパズル | Lv1-8 (形合わせ+2D回転) | Lv14+ (系列補完) | 視覚マッチ正答率 ≥ 80% + MA ≥ 5 |
| たんけんめいろ | Lv1-13 (迷路) | Lv14+ (順序計画) | 迷路完遂可能 + 順方向span ≥ 3 |
| ことばとえほん | Lv1-8 (基本語彙) | — | 音声1段階指示理解あり（※重度IDはロック） |
| きもちフレンズ | Lv1-8 (マッチ形式) | Lv19+ (情動Go/No-Go) | 顔一致課題成立 + Go/No-Go成立 |
| タッチアドベンチャー | Lv1-13 | Lv14+ (精密操作) | 中ターゲットタップ成立 |

### 重度ID 3歳児が意味を持ってプレイできるゲーム数

**8本中、実質3〜4本**:
- ひかりレスキュー (Lv1-3: 触ると光る/消える)
- タッチアドベンチャー (Lv1-3: 大ターゲットタップ)
- たんけんめいろ (Lv1-3: 一本道スワイプ)
- おぼえてすすむ (Lv1-3: 順方向2要素、デモ支援前提)

### 知的に正常なASD/ADHD 6歳児の退屈防止

- 上位難度の早期解放（キャリブレーションで即Tier 3判定可能に）
- 刺激テーマ選択（宇宙/海/森などのスキン）
- AI推薦 + 2択の自己選択余地（choice opportunityがon-task behaviorを向上）

## 5. 指示方法: 4階層設計

| レベル | 対象 | 指示形式 | 実装要件 |
|--------|------|---------|---------|
| **L1: 非言語** | 重度ID、3-4歳、言語理解限定 | デモアニメのみ。手本キャラが正解をタップする様子を再生。言葉なし | テキストなし、音声なしで成立すること。誤答時は「無反応」（ブザー音はパニック誘発リスク） |
| **L2: 最小言語** | 中度ID、低年齢 | デモ + アイコン + 1語音声（「タップ」「まって」等） | 1語1命令。文字表示なし |
| **L3: 音声支援** | 軽度ID、幼児一般 | デモ + 短文音声 + アイコン（「あかいのを タッチしてね」） | 1文1命令、二重否定禁止、ひらがな主体、漢字にルビ |
| **L4: 視覚優位対応** | 知的正常ASD、学齢期 | デモ + テキスト + 音声ON/OFF | 音声オフ可能（聴覚過敏対応）、手順固定表示 |

### 指示レベルの決定方法

- オンボーディングの言語理解回答から初期設定
- 実プレイ中の理解度（ランダム押し率、デモ再視聴回数）で動的調整
- 保護者が手動変更可能

### 言語なしでルール理解できるゲーム

- **可能**: ひかりレスキュー(Lv1-8), おぼえてすすむ(順方向), くるくるパズル(形合わせ), たんけんめいろ, タッチアドベンチャー
- **条件付き可能**: ルールチェンジ（デモを丁寧にする必要）, きもちフレンズ(マッチ形式)
- **不可能/非効率**: ことばとえほん（音声理解が本体）

## 6. DDA (Dynamic Difficulty Adjustment) 仕様

### 障害種別ターゲット正答率

直接的なRCT根拠は存在しないが、ZPD理論・competence support・幼児frustration管理の知見、およびABAマスタリー基準（Granpeesheh et al., 2014）、Errorless Learning原則（PubMed 31901670）から以下を推奨:

| 特性 | ターゲット正答率 | 根拠 |
|------|----------------|------|
| ASD（失敗回避傾向） | 80-90% | ABAマスタリー基準80-100%×2-3連続セッション（Leaf & McEachin, 1999）。失敗への耐性が低い。成功体験重視 |
| ADHD（即時報酬感度高） | 70-85% | Flow理論との整合。刺激不足で飽きる。適度な挑戦が必要。EndeavorRx臨床試験でも適度な挑戦難易度が効果の核心 |
| ID 重度 | **90-95%** | **Errorless Learning原則**。エラー経験は誤概念の固着リスクがあり高成功率設計が必須（Tandem & Koegel, 2015）。学習性無力感防止 |
| ID 軽度-中度 | 80-90% | ASDに準じる |
| 測定フェーズ | 70-80% | 天井・床効果回避のため |

> **v2からの変更**: ID重度の下限を85%→90%に引き上げ。根拠: Errorless Learning研究では85%でもエラー頻度が高すぎる可能性が示唆されている（PMC5843573）。ID重度児は「できないことを繰り返す」体験がanxiety・problem behaviorの引き金になるリスクがある

### 試行間インターバル（ITI）の障害別設定

> **v3新設**: リサーチで障害種別ごとの処理速度差が明確になったため追加

| 障害種別 | ITI | 根拠 |
|---------|-----|------|
| ADHD | 1.0〜1.5秒 | 速いペースで注意維持。遅延が退屈→ドロップアウトの連鎖を引き起こす |
| ASD | 1.5〜2.0秒 | 処理時間の余裕を確保。予測可能なリズムが重要 |
| ID 軽度 | 2.0〜2.5秒 | 処理速度低下への配慮 |
| ID 重度 | 3.0秒以上 | 処理速度の大きな違いを考慮。Swellerの認知負荷理論（1988）に基づく |

実装: `DDA_CONFIG`にITI設定を追加。ユーザープロファイルの障害種別に連動して自動適用

### DDA共通ルール

| ルール | 仕様 | 根拠 |
|--------|------|------|
| 難化条件 | 2連続成功窓で1段階UP | 急激な難化を避ける |
| 易化条件 | 1窓逸脱で即時1段階DOWN | 挫折を最小化 |
| 窓サイズ | ゲーム依存（Go/No-Go: 8-10試行、Corsi: 2-3系列、迷路: 1-2面） | 課題特性に合わせる |
| パラメータ優先順 | **弁別性 → 刺激数 → 速度 → 時間制限** | 時間制限は失敗感を増やしやすい。学習コスト最小化順 |
| 天井検出 | 3セッション連続最高レベル → 上位ティアゲーム推薦 | 退屈防止 |
| 床検出 | 3セッション連続最低レベル → 指示レベルDOWN or 保護者通知 | 課題不適合の早期発見 |

### フラストレーション検出（既存FrustrationDetector拡張）

- **連続誤答**: 3連続で即時易化（窓完了を待たない）
- **無反応**: 10秒無操作で「ヒント表示 or デモ再生」
- **連打**: 1秒内3タップ以上 → 衝動連打と判定、試行無効化
- **離脱**: 30秒無操作 → セッション中断提案

## 8. 価値定義の転換

### Before（現行の暗黙前提）
「ゲームで認知機能を鍛える（far transfer）」

### After（推奨）
「ゲームを通じて認知特性を理解し、得意を発見し、近接スキルを改善する」

### Near Transfer / Far Transfer 方針（v3強化）

> **根拠**: メタ分析（Valeri et al., 2023, PMC10920464）で「日常生活への転移効果（Far Transfer）」のエビデンスは限定的。第二次メタ分析（Sala & Gobet, 2019, Collabra）では全体的なFar Transfer効果はほぼゼロ（d ≈ 0）

**Manasの公式ポジション**:
- **Near Transferは主張する**: 各ゲームが測定・訓練するスキル（WM・抑制・注意持続等）の内部改善は達成可能。これは複数の競合RCTで実証済み
- **Far Transferは主張しない**: 「学校の成績が上がる」「IQが上がる」「発達障害が治る」は現時点のエビデンスで支持されない
- **自己効力感・自己理解は主張する**: Strengths-Based Approachの研究（Taylor et al., 2023, SAGE）で、強みの活用がQOL・精神的well-being・メンタルヘルスを有意に改善することが実証されている
- **Lumosity教訓**: 2016年FTC罰金$2M。「脳トレで認知症予防」等の過剰効果主張が原因。Manasは同じ轍を踏まない

### 保護者向けメッセージング
- ✅ 「お子さまの注意力・記憶力・柔軟性のプロファイルが見えてきます」
- ✅ 「苦手な部分を練習し、得意な部分を伸ばします」
- ✅ 「遊びを通じて少しずつできることが増えていきます」
- ✅ 「お子さまの得意なことを一緒に発見しましょう」（Strengths-Based）
- ❌ 「IQが上がります」
- ❌ 「学力が向上します」
- ❌ 「発達障害が治ります」
- ❌ 「学校での成績が改善します」（Far Transfer主張）

### 投資家向けメッセージングとの整合
- 投資家ピッチでは「Duolingo for neurodivergent」を訴求するが、これはエンゲージメント設計の方法論を指しており、効果の主張ではない
- 「鍛える中身はCogniFit、続けさせる仕組みはDuolingo」は設計哲学の説明であり、効果の保証ではない
- エビデンスロードマップ（Section 17）で段階的にRCTを実施し、主張範囲を拡大する計画を提示

## 9. セッション設計: 混合セッション型

### 原則

1セッション内で**複数の異なるゲームを混合**する。Duolingo型の「フォーマット切替」設計。

**根拠:**
- 注意の切り替え自体が認知トレーニングになる（Diamond, 2013）
- 単調さの排除がADHD児の注意維持に有効
- CogniFitのITS（Individualized Training System）も混合セッション設計
- Duolingoの成功要因の核心は「1レッスン内で問題形式が毎回変わる」点

### セッション構成

**1セッション = 3ゲーム × 各2-3試行 + ウォームアップ**

| ティア | 1セッション長 | ゲーム数 | 各ゲーム試行数 | 合計試行数 |
|--------|-------------|---------|-------------|----------|
| Tier 1（MA < 2歳） | **1.5-2分** | 2ゲーム | 2試行 | 4 + ウォームアップ1 |
| Tier 2（MA 2-4歳） | **2-3分** | 3ゲーム | 2試行 | 6 + ウォームアップ1 |
| Tier 3（MA > 4歳） | **3-5分** | 3ゲーム | 3試行 | 9 + ウォームアップ2 |

### ゲーム選択ロジック（SessionEngine）

1セッションに含めるゲームの選択:

1. **弱いドメインのゲームを優先** — 認知プロファイルから最も伸びしろのあるカテゴリのゲームを選択
2. **同一カテゴリを連続させない** — 注意・抑制→記憶→柔軟性のように異なるカテゴリから選択
3. **前回セッションと異なるゲームを優先** — 多様性確保
4. **ティアのロック制限を尊重** — ロックされたゲームは選択肢に含めない

**ASD向け配慮**: ゲーム順序を毎回ランダムにしない。「A→B→C」の順序パターンを数回繰り返した後に変更する（予測可能性の確保）

### セッション進行フロー

```
[ウォームアップ] → [ゲームA 試行1] → [ゲームA 試行2] 
  → [トランジション演出（キャラが移動）]
  → [ゲームB 試行1] → [ゲームB 試行2]
  → [トランジション演出]
  → [ゲームC 試行1] → [ゲームC 試行2]
  → [セッション完了画面]
```

### ユニット制（5セッション = 1ユニット）

| セッション# | 内容 | 目的 |
|------------|------|------|
| 1 | 混合セッション | トレーニング |
| 2 | 混合セッション | トレーニング |
| 3 | **ボーナスチャレンジ（ミニ報酬）** | 中間達成感 |
| 4 | 混合セッション（苦手要素重点化） | 間隔反復 |
| 5 | **ユニットレビュー（ボス戦）** | 統合評価＋大報酬 |

### 1日のプレイ上限

上限は**累計プレイ時間のみ**で制御。セッション間クールダウンは設けない。

| ティア | 1日の最大プレイ時間 |
|--------|-------------------|
| Tier 1 | 10分 |
| Tier 2 | 20分 |
| Tier 3 | 30分 |

上限到達時: 「きょうは たくさん がんばったね！また あした あそぼう 🌟」

### 日による認知状態への適応

セッション開始時に**ウォームアップ**（採点対象外）を実施。最初のゲームの簡単な試行で前回レベルとの比較:
- 全問正解 → 前回レベルから開始
- 半分不正解 → 前回レベル-1から開始
- 全問不正解 → 前回レベル-2から開始

## 10. フィードバック設計

### 感覚過敏コントロールUI（v3新設）

> **根拠**: ASDの37〜69%が聴覚過敏（hyperacusis）を持つ（UX Magazine, 2024）。WCAG 2.2 SC 1.4.11準拠。W3C COGA Design Patternで感覚過敏ユーザーへの調整可能性が推奨

**設定画面に以下のユーザーコントロールを追加**:

| 設定項目 | オプション | デフォルト | 根拠 |
|---------|----------|---------|------|
| BGM | オン / オフ | オン（音量50%） | BGMはADHD/ASDの集中を妨害する可能性 |
| 効果音の音量 | 大 / 普通 / 小 / オフ | 普通 | アタック20ms以上、リリース200ms以上。440〜880Hzの中音域に限定 |
| アニメーション速度 | ふつう / ゆっくり / なし | ふつう | prefers-reduced-motion準拠。ASD・ADHD両者に推奨 |
| 画面の明るさ | ゲーム内調整スライダー | システム依存 | 視覚過敏への配慮 |
| 振動フィードバック | オン / オフ | オン | 触覚過敏への配慮 |

**色彩設計ルール**:
- 赤色不使用（実装済み）
- 高彩度色の上限: 彩度60%以下（CSS変数で管理）
- フラッシュ・点滅: WCAG 2.2準拠で3回/秒以上の点滅禁止
- コントラスト比: WCAG AA 4.5:1以上

**実装**: ゲームオプション画面（歯車アイコン）から一括設定。`useSettings` hookで全ゲームに反映

### 移行期予告UI（v3新設）

> **根拠**: ASD児の移行困難特性。IIDC (Indiana University, 2023)のASD移行支援ガイドライン

| タイミング | 表示内容 | 形式 |
|-----------|---------|------|
| セッション終了3分前 | 「もうすぐおわるよ」 | トースト表示 + ビジュアルカウントダウン（5→4→3→2→1） |
| ゲーム切り替え時 | 「つぎは ○○ だよ」+ 次ゲームの絵カード | 3秒間の予告画面 → 自動遷移 |
| セッション終了 | 「またあした！よく がんばったね」| カレンダービジュアル + 植物の今日の成長 |

### 正解フィードバック: 多感覚条件付け

正解時のフィードバックは**古典的条件付け**の原理を活用し、「成功 = 快」の神経回路を強化する。

| モダリティ | 仕様 | 感覚過敏対応 |
|-----------|------|------------|
| **聴覚** | 短い高音「ding」（200ms以内）。ADHDにはリズミカルなテンポ | 音量5段階調整 + 完全OFF可能 |
| **視覚** | ターゲットが光る＋キャラクター笑顔アニメ（300ms） | エフェクト強度3段階（控えめ/標準/派手） |
| **触覚** | 軽い振動パルス（50ms） | ON/OFF切替可能 |

**連続正解ブースト**: 
- **3問連続正解**でマスコットが喜ぶアニメーション（Duolingoの5問を短縮。ADHDの即時報酬感度に適合）
- **5問連続正解**でスペシャルエフェクト（コンフェッティ等）
- 連続ブーストの視覚強度はティアで調整（Tier 1は控えめ、Tier 3は派手）

### 不正解フィードバック: 挫折ゼロ設計

| 状況 | フィードバック | 絶対にやらないこと |
|------|-------------|-----------------|
| 不正解（1回目） | 無音 + 画面が軽く揺れる + 「もう一回やってみよう」 | ブザー音、バツ印、赤色全面表示 |
| 不正解（2回目） | ヒント表示（正解を強調エフェクト）+ 「こっちだよ！」 | 「ざんねん」「まちがい」等のネガティブワード |
| 不正解（3回目） | デモ再生（正解をキャラが実演）→ 同じ問題をもう一度 | スコア大幅減算、失敗カウンター表示 |
| 惜しい不正解 | **「おしい！」メカニクス**: 連続正解ブーストを失わずにリトライ可能 | — |

**科学的根拠**: 
- 不正解音（ブザー等）は重度ID児にパニックを誘発するリスク（レビュー1）
- 成長マインドセット研究では、失敗を「学習の一部」として肯定的にフレーミングすることが動機づけを維持（Dweck, 2006）
- 「惜しい！」メカニクスはnear-miss効果を利用し、次の試行への動機を高める

### スキャフォールディング（段階的支援）

同じ認知機能の問題で2セッション連続して低成績（ターゲット正答率を10%以上下回る）の場合:

1. **ヒント頻度UP**: 不正解1回目からヒント表示
2. **選択肢数DOWN**: 4択→3択→2択
3. **デモ頻度UP**: 試行開始前に自動デモ再生
4. **DDA即時易化**: 通常の窓完了を待たず1段階DOWN

## 11. 報酬・継続性設計

### 報酬の多層構造

| 層 | トリガー | 報酬内容 | 頻度 |
|----|---------|---------|------|
| **マイクロ報酬** | 毎正解 | ding音 + 視覚エフェクト + 振動 | 毎試行 |
| **連続報酬** | 3問/5問連続正解 | キャラアニメ / スペシャルエフェクト | セッション内数回 |
| **セッション完了報酬** | 1セッション完了 | ★獲得 + 3択宝箱（下記参照）+ プログレス進行 | 毎セッション |
| **ユニット完了報酬** | 5セッション完了 | ボス戦クリア演出 + コレクティブル獲得 | 5セッションごと |
| **マイルストーン報酬** | 累計達成 | 新キャラ解放 / 新テーマ解放 / 特別バッジ | 不定期 |

### 宝箱メカニクス（可変報酬）

セッション完了時に**3つの宝箱**から1つを選ぶ。中身はランダム。

- コレクティブルアイテム（キャラの衣装、背景テーマ）
- ボーナススター
- レアアイテム（低確率で特別エフェクト）

**ASD対応オプション**: 「いつもの報酬モード」ON → 宝箱なし、毎回同じ報酬（予測可能性を優先）

### ストリーク設計: 植物メタファー

**Duolingoのストリーク（完全リセット）は障がい児に不適切**。体調不良、通院、パニック等でプレイできない日が頻繁にあるため。

**植物成長システム**:

```
成長レベル = 連続プレイ日数に応じて1〜5段階で成長
プレイした日: 成長 +1（最大5）
プレイしなかった日: 成長 -1（最低1。種に戻る。ゼロ＝消滅はしない）
```

| 成長レベル | 表示 | 報酬 |
|-----------|------|------|
| 🌱 1（種） | 土から芽が出かけ | 基本XP |
| 🌿 2（芽） | 小さな双葉 | XP × 1.2 |
| 🌷 3（花） | 花が咲く | XP × 1.5 + ボーナスアイテム |
| 🌳 4（木） | 立派な木に成長 | XP × 2.0 + レアアイテム確率UP |
| 🌲🌳🌴 5（森） | 森が広がる | XP × 2.5 + 限定報酬 |

**核心**: 
- 1日休んでも森→木に戻るだけ（枯れない）
- 3日休んでも芽が残っている
- 「育てた植物が死ぬ」恐怖がない（最低でも種に戻るだけ）
- 復帰時は「おかえり！お水をあげよう 💧」のポジティブメッセージ
- 子どもにとって植物の成長は直感的に理解しやすい

### ストリーク保護バッファ（v3新設）

> **根拠**: Duolingo Streak Freezeの導入でDay14リテンション+14%。7日ストリーク到達後の保護機能が長期リテンションの鍵（Sensor Tower, 2024）

**設計**:
- 7日以上のストリーク保持者に「週1回の猶予日」を自動付与
- 猶予日使用時: 成長レベルは-1されない（成長停止のみ）
- 猶予日を使った翌日: Lunaが「おやすみしたけどストリーク続いてるよ！」と通知
- 猶予日の事前説明: Lunaが「今週は1回お休みができます」と毎週月曜に表示

```typescript
interface DailyStreak {
  currentDays: number;
  growthLevel: 1 | 2 | 3 | 4 | 5;
  lastPlayedDate: string;
  graceAvailableThisWeek: boolean;
  graceUsedThisWeek: boolean;
}
```

**通知との連携**:
- 「お花がお水を待っています 🌷」（デイリーリマインダー）
- 「すごい！森ができたよ！🌲🌳🌴」（MAX達成時）

## 13. 間隔反復（Spaced Repetition）

### 原理

Duolingoの半減期回帰モデル（HLR）を認知トレーニングに適応。「忘れかけた瞬間」に復習することで定着を最大化する。

### Manasでの適用

| 要素 | 仕様 |
|------|------|
| **苦手パターンの再出題** | セッション内で不正解だった問題を、同セッション末尾で再出題 |
| **セッション間の間隔反復** | 不正解パターンを1日後 → 3日後 → 7日後に再出題 |
| **障害特性別の反復間隔** | ID: 間隔を短く（1日→2日→4日）、ADHD: 標準、ASD: 間隔は標準だが同一フォーマットで出題 |
| **ユニットレビュー（ボス戦）** | 5セッション目で過去4セッションの苦手要素を集約して再テスト |

### 反復間隔の初期パラメータ

```
base_interval = 1日
multiplier_on_success = 2.0
multiplier_on_failure = 0.5
max_interval = 14日（2週間以上空くと再学習が必要なため）

障害別補正:
  ID: base_interval × 0.7, max_interval × 0.5
  ADHD: base_interval × 1.0（標準）
  ASD: base_interval × 1.0, ただし刺激フォーマット固定
```

## 15. オンボーディングとキャリブレーション

> **v3抜本改訂**: リサーチにより、v2の25画面フローはTTV（価値実感時間）が業界最低水準と判明。Duolingoがサインアップ後置で翌日リテンション+20%を達成した事例を踏まえ、「3画面でゲーム体験→サインアップは後置」に再設計。認知ドメイン質問8問はDDAのゲームプレイデータで代替し削除

### フロー設計（v3）

```
【Phase 1: 高速開始】（2分・3画面）
  [Screen 1] 生年月日選択 → AgeGroup自動判定
  [Screen 2] 名前入力（ニックネームOK）
  [Screen 3] 発話レベル選択（アイコン付き5択）
      ↓
【Phase 2: 初回ゲーム体験 = キャリブレーション】（3分）
  マスコット登場アニメ（10秒）
  ├─ 単純反応（タッチアドベンチャーLv1）→ 運動能力ベースライン
  ├─ Go/No-Go簡易版（ひかりレスキューLv4）→ 抑制ベースライン
  ├─ 図形マッチ（くるくるパズルLv1）→ 視空間ベースライン
  └─ 結果: 「よくできました！○○くんの得意なこと、わかってきたよ」
     認知プロファイルのシードデータをバックグラウンドで生成
      ↓
【Phase 3: アカウント作成】（任意・1分）
  「進捗を保存して、○○くんのプロフィールを完成させましょう」
  （スキップ → 匿名セッションとして継続可能）
      ↓
【Phase 4: ホーム画面到着】
  ティアに応じたゲームが解放された冒険マップ表示
      ↓
【Phase 5: 任意の詳細情報】（後置・保護者ダッシュボードから）
  診断情報・行動特性・評価歴
  → 「入力すると○○のゲームが最適化されます」という動機付け
  → ゲームプレイ3回目以降に「もっと合わせるために教えて」として誘導
```

- **TTV: 約5分**（v2では25画面完了まで推定10〜15分）

- サインアップは後置。匿名セッション→後日アカウント作成の導線を用意

| 旧Screen | 情報 | Phase 5での扱い |
|---------|------|---------------|
| Screen 3 | 発話レベル | Phase 1に統合（Screen 3） |

### キャリブレーション → ティア判定ロジック

```
if (全3試行でランダム押し or 無反応) → Tier 1
else if (単純反応OK かつ Go/No-Go不成立 or 図形マッチ不成立) → Tier 2

  - 診断「重度ID」→ Tier上限を2に制限
  - 診断「中度ID」→ Tier3のStop-signal/backwardをロック
  - 年齢 < 4歳 → Tier3の一部をロック
```
