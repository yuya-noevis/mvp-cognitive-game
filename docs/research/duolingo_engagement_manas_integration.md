# Duolingoエンゲージメント設計分析 → Manas統合戦略

## コンセプト：CogniFitの「科学」× Duolingoの「習慣エンジン」

Manasの設計哲学を一言で言うと：**「鍛える中身はCogniFit、続けさせる仕組みはDuolingo」**

CogniFit = 600万ユーザー（科学は強いが、続かない）
Duolingo = 1.13億MAU（続くが、認知科学の深さはない）
Manas = 両方の強みを統合した、神経発達障害児向けの唯一のプロダクト

## Part 1：Duolingoのセッション設計 — 数字で見る

### 1-1. レッスンの基本構造

| 要素 | Duolingoの設計 | Manas への適用案 |
|------|---------------|----------------|
| **1レッスンの問題数** | 15問（固定ベース。誤答すると追加問題） | 8〜12問（ID児の注意持続時間に合わせ短縮） |
| **1レッスンの所要時間** | 3〜5分（簡単なレッスン）〜10分（難しいレッスン） | 3〜5分を厳守。最大7分 |
| **推奨日次学習時間** | 15分/日（公式の最適値。社内研究で確立） | 10〜15分/日（2〜3セッション） |
| **問題タイプの多様性** | 1レッスン内に5〜6種類（選択、並べ替え、リスニング等） | 1レッスン内に3〜4種類（タップ、ドラッグ、タイミング等） |
| **進捗バー** | レッスン中に常時表示。正答で進む、誤答で戻る | 同様に実装。ただし「戻る」は最小限に（成功体験重視） |
| **誤答時の挙動** | 即座にフィードバック→正解表示→追加問題で再出題 | 即座にフィードバック→視覚ヒント表示→再試行（正解を出すまで） |

### 1-2. Duolingoが発見した重要な数値的知見

**「短くしすぎると逆効果」：** Duolingoはレッスンを短縮する実験を行ったが、TSLW（Time Spent Learning Well）が低下した。セッションは「短すぎず、長すぎず」のスイートスポットがある。

**「1日1レッスン」がストリークの最適単位：** 当初XPベース（例：10XP稼ぐ）だったストリーク条件を「1レッスン完了」に変えたところ、DAUが大幅に増加。さらに「1問だけ」に下げる実験は失敗した — ユーザーの満足感と習慣形成に十分なコミットメントが必要。

**「15分/日がスイートスポット」：** 社内研究と外部研究で確立。Duolingoは全ユーザーの15分/日到達率を最適化指標としている。

→ **Manas設計指針：** 1セッション=8〜12問、3〜5分。1日の目標を「1セッション完了」に設定。親子で一緒にやる場合は2〜3セッション（10〜15分）を推奨。セッションの最小単位は十分な達成感がある量（1問では少なすぎる）。

### 1-3. セッション内の問題タイプローテーション

Duolingoは1レッスン内で問題タイプを頻繁に切り替えることで飽きを防ぐ：

```
[選択問題] → [並べ替え] → [リスニング] → [翻訳] → [穴埋め] → ...
```

同じスキルを、異なるモダリティ（視覚、聴覚、産出）で繰り返し訓練する設計。

→ **Manas設計指針：** 同じ認知スキル（例：ワーキングメモリ）を、異なるゲームメカニクスで1セッション内に3回以上触れさせる：

```
[光る丸を覚えてタップ] → [迷路で3手先を計画] → [パターンの続きを選択]
```

これにより「飽き」を防ぎつつ、同じ認知機能への反復刺激を確保。

## Part 2：Duolingoのリテンション・エンジン — Manasへの変換

### 2-1. ストリーク（最重要機能）

**Duolingoの数値：**
- ストリークはDAU成長の最大のドライバー
- 7日ストリーク維持者は長期継続率が3.6倍
- ストリークフリーズ導入でチャーン21%削減
- iOSウィジェット（ストリーク表示）で日次利用率60%増
- 9百万人以上が1年以上のストリークを維持
- ストリーク機能だけで600以上のA/Bテストを実施

**Manas版ストリーク設計：**

| Duolingo | Manas（子ども向け適応） |
|----------|----------------------|
| 🔥 火のアイコン＋日数 | 🌱 成長する植物（種→芽→花→木→森）ストリーク日数で成長 |
| ストリークフリーズ（有料） | おやすみチケット（無料で週1回。有料で追加購入） |
| ストリーク失敗→ゼロリセット | ストリーク保護期間（3日間で回復可能。完全リセットは避ける） |
| 自己アイデンティティ化 | 「〇〇ちゃんの木が△日育ったよ！」保護者への共有通知 |

**重要な設計原則：** Duolingoの「ストリークリセット」は損失回避で強力だが、**神経発達障害児の保護者にとっては「子どもが失敗した」感覚を生む**リスクがある。Manasでは**完全リセットを避け、「回復可能なストリーク」**を採用する。植物メタファーなら「枯れかけ→水やりで復活」が自然。

### 2-2. XP（経験値）システム

**Duolingoの設計：**
- レッスン完了でXP獲得（通常10〜15XP）
- 連続正答でコンボボーナス
- XPブースト（2倍、3倍）が限定時間で出現
- デイリーゴール（10/20/30/50 XP/日から選択）

**Manas版XP設計：**

| 要素 | Manas設計案 |
|------|-----------|
| XP名称 | 「ひかりポイント」（Manasのブランドカラー紫に合わせ、星や光のモチーフ） |
| 獲得トリガー | セッション完了、連続正答、新しいゲーム初クリア、難易度レベルアップ |
| デイリーゴール | ふつう（1セッション）/ がんばる（2セッション）/ チャレンジ（3セッション）|
| ボーナス | 連続正答5問でキラキラエフェクト＋ボーナスポイント |
| レベルアップ | 一定ポイントでレベルアップ→新しいアバターアイテムや背景アンロック |

### 2-3. リーダーボード／リーグ

**Duolingoの数値：**
- リーグ導入でレッスン完了率25%増
- ソーシャル比較が週あたり15%多くのレッスン完了を駆動
- 10段階のリーグ（ブロンズ→ダイヤモンド）で週次昇降格

**Manas版（慎重な適応が必要）：**

神経発達障害児のコンテキストでは**競争的リーダーボードは不適切**な場合が多い。代替設計：

| Duolingo | Manas適応 |
|----------|----------|
| 個人ランキング（XP順位） | 「みんなのもり」— 全ユーザーの植物（ストリーク）が一つの森を構成。協力型 |
| 週次リーグ（昇降格） | 「今週のチャレンジ」— 個人目標の達成率で★獲得。他者との比較なし |
| フレンドクエスト | 「おやこクエスト」— 保護者と一緒にクリアするミッション |

→ **競争ではなく協力と自己ベスト更新を軸にする。** ただし、ADHD児や知的に高いASD児には任意でオプトインできるチャレンジモードを用意。

### 2-4. 通知・リマインダー

**Duolingoの数値と戦略：**
- プッシュ通知がDAU成長の重要ドライバー
- アプリアイコンに赤バッジ（未完了表示）追加でDAU+6%
- 「ストリークが途切れそうです！」通知がリテンションに直接貢献
- 過度な通知は5%のユーザー不満を引き起こし、オプトアウト設計を追加
- 通知のコピーライティングA/Bテスト：「Continue」→「Commit to My Goal」変更で大幅改善
- 成長マインドセット的な応援メッセージ（「がんばってるね！」）でD14リテンション+7.2%

**Manas版通知設計（保護者向け）：**

子ども自身がスマホを持たないケースが多いため、**通知の受信者は保護者**：

| タイミング | メッセージ例 | 心理的効果 |
|-----------|-----------|----------|
| 毎日の練習時間前 | 「〇〇ちゃんの練習時間です！🌱 今日やったら5日連続！」 | ルーティン形成 |
| ストリーク危機 | 「〇〇ちゃんの木が少し元気がないみたい。今日5分だけ水やりしませんか？」 | 損失回避（温かいトーン） |
| 達成時 | 「〇〇ちゃん、ワーキングメモリが先週より↑！すごい！」 | 正の強化（具体的成長の可視化） |
| 週次レポート | 「今週のまとめ：集中力⬆️ 記憶力→ 計画力⬆️」 | エビデンスベースの進捗報告 |

**重要：** Duolingoの「Duoの悲しい顔」的な罪悪感マーケティングは、障害児の保護者にとっては有害。Manasでは**常にポジティブ・温かい・支援的なトーン**を維持する。

### 2-5. マイクロウィン（小さな達成感）

**Duolingoの設計：**
- 正答時：緑チェックマーク＋効果音＋短いアニメーション
- レッスン完了時：花火エフェクト＋XP獲得表示＋「Session Complete!」
- Duoの手を振るアニメーション
- 完了演出を即座に表示し、バックグラウンド処理は裏で実行（体感遅延60%削減→DAU増加）

→ **これはManasにそのまま適用すべき最重要要素の一つ。** 正答時の即時フィードバック（音＋視覚）と、セッション完了時の派手な演出は、神経発達障害児にとって**強化子（reinforcer）として機能**する。ABA（応用行動分析）の即時強化の原則と完全に一致する。

**Manas版マイクロウィン：**

| タイミング | 演出 | 配慮 |
|-----------|------|------|
| 正答時 | キラキラエフェクト＋肯定音＋キャラクター笑顔 | 音量調整可能。視覚エフェクトの強度調整可能（感覚過敏対応） |
| 連続正答 | コンボカウンター＋エフェクト増加 | 5連続で特別演出 |
| セッション完了 | 花が咲く／星が集まるアニメーション＋ポイント獲得 | 即座に表示（バックグラウンド処理） |
| レベルアップ | 全画面演出＋新アイテムアンロック | 保護者にも通知（一緒に喜べる） |
| 週次マイルストーン | 「今週のがんばり」振り返りカード | 保護者と共有可能 |

### 2-6. ハートシステム（失敗のコスト設計）

**Duolingoの設計：**
- 5つのハート（ライフ）でスタート
- 誤答1回でハート-1
- ハートゼロでレッスン不可→4時間で1ハート回復、またはジェムで購入
- Practice Hub（復習）でハート回復可能
- 有料会員は無制限ハート

**Manas版（大幅に変更が必要）：**

Duolingoのハートシステムは「失敗のコスト」を設けることで集中力を高めるが、**神経発達障害児にとっては以下のリスク**がある：
- 失敗の恐怖→回避行動→アプリを開かなくなる
- ハートゼロ→フラストレーション→かんしゃく
- 「間違えてはいけない」プレッシャー→学習への負の連合

→ **Manasではハートシステムを採用しない。** 代わりに：

| 概念 | Manas設計 |
|------|----------|
| 失敗のコスト | なし。何度でも再試行可能 |
| 難易度調整 | 誤答が続くと自動的に難易度が下がる（ユーザーには見えない） |
| 正答の報酬 | 正答でボーナスポイント（加算のみ、減算なし） |
| 集中力の維持 | ハートの代わりに「チャレンジスター」— 1セッション内の正答率で★1〜3個獲得。失っても次回に影響なし |

## Part 3：Duolingoのオンボーディング → Manasへの変換

### Duolingoのオンボーディング設計原則

1. **サインアップ前にレッスン開始** — 最初の「できた！」を数分以内に体験させる
2. **コミットメントデバイス** — 目標設定（1日何分？）を聞くことで自己拘束
3. **パーソナライゼーション感** — 言語選択、理由、レベルを聞いて「自分用」感を演出
4. **初回レッスンは極端に簡単** — 100%正答可能なレベルから開始

**Manas版オンボーディング：**

```
Step 1: 保護者がアプリを開く
Step 2: 「お子さまの年齢は？」「お子さまの特性は？」（ASD/ADHD/ID/その他）
Step 3: 「まずは試してみましょう！」→ 子どもに渡す
Step 4: 超簡単なミニゲーム（画面タップで星が光る等）→ 即座に成功体験
Step 5: 「すごい！〇〇ちゃんが最初のひかりポイントを獲得！」
Step 6: ここでアカウント登録
Step 7: 初回アセスメント（ゲーム形式、5分）→ 認知プロファイル生成
Step 8: 「〇〇ちゃん専用のトレーニングプランができました！」
```

→ Duolingoの「価値を先に体験させてからサインアップ」の原則を踏襲しつつ、保護者と子どもの二層構造に適応。

## Part 4：Duolingoの実験文化 → Manasの成長エンジン

### Duolingoの数値

- ストリーク機能だけで600以上のA/Bテスト（4年間で）
- 2024年にAndroidパフォーマンスチームだけで200以上のA/Bテスト
- ボタンテキスト変更1つで+10,000 DAU
- アプリアイコンの赤バッジ追加で+6% DAU
- DAUを唯一のNorth Star指標として全チームが最適化
- CURR（Current User Retention Rate）をDAUの先行指標として使用

### Manasへの適用

初期段階では大規模A/Bテストは不可能だが、**実験マインドセット**は最初から組み込める：

| Duolingoの原則 | Manas初期フェーズでの適用 |
|---------------|----------------------|
| DAUがNorth Star | 「週次アクティブセッション数」をNorth Starに |
| CURR（翌日復帰率）が先行指標 | 「翌日セッション完了率」を追跡 |
| コピーのA/Bテスト | 通知メッセージのバリエーションテスト（少人数でも可） |
| 機能テスト→DAU影響測定 | 新ゲーム追加時に完了率・復帰率を測定 |
| 小さなテストが大きな影響 | UIの微修正（ボタン色、サウンド、アニメーション）を継続的に調整 |

## Part 6：Duolingoから学ぶ「やってはいけないこと」

Duolingoの設計にも課題がある。Manasのコンテキストでは特に注意すべきポイント：

### 6-1. 過度な損失回避の利用
Duolingoの「Duoの悲しい顔」通知や「ストリークが途切れます！」は、健常成人には効果的だが、障害児の保護者には**罪悪感と疲弊**を引き起こす。Manasでは常にポジティブなフレーミングを使う。

### 6-2. 競争的リーダーボードの無批判な導入
Duolingoのリーグシステムは効果的だが、発達障害児が他児と比較されることへの保護者の懸念は非常に強い。Manasでは**協力型・自己ベスト更新型**のみを標準とする。

### 6-3. ハートシステム（失敗のペナルティ）
前述の通り、Manasでは不採用。

### 6-4. 過密な通知
Duolingoは通知の過剰さで5%のユーザー不満を引き起こした。療育中の保護者は多忙であり、通知は**1日最大2回**に制限し、全てオプトアウト可能にする。

### 6-5. 課金圧力
Duolingoのジェム購入やSuper Duolingoへの導線は、障害児の保護者（多くが経済的負担を抱えている）には不適切に映る可能性がある。Manasの課金導線は**控えめで、無料枠でも十分な価値を提供**する設計とする。
