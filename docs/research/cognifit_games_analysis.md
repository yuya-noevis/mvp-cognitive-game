# CogniFit ゲーム・タスク詳細分析：Manasゲーム設計への示唆

## はじめに

CogniFitは60以上のトレーニングゲームと17の評価タスクで構成されている。各ゲームは2〜4つの認知スキルにマッピングされ、各評価タスクは確立された神経心理学的検査をデジタル化したものである。このレポートでは、ゲームの具体的内容を分類・分析し、Manasのゲーム設計に直接活用できる知見を整理する。

---

## Part 1：17の評価タスク（CAB）の詳細

各タスクは古典的な神経心理学的検査に基づいて設計されており、CogniFitの科学的信頼性の根幹をなしている。

### 1-1. 記憶系タスク

| タスク名（コード） | 内容 | 元になった検査 | 測定スキル |
|---|---|---|---|
| **数列テスト（WOM-ASM）** | 画面に表示される数字の列を記憶し、同じ順番で再現。列は徐々に長くなる | WAIS-III 数唱（Wechsler, 1997） | 短期記憶、音韻性短期記憶、ワーキングメモリ、処理速度 |
| **認識テスト（WOM-REST）** | 3つのオブジェクトの組み合わせを記憶し、次の画面で4つの選択肢から同一のものを選ぶ | WAIS 記号探索 + WCST + Ravenマトリクス | 空間知覚、視覚走査、再認、非言語記憶、ワーキングメモリ |
| **集中力テスト（VISMEM-PLAN）** | 固定配置の円のうち光った円を記憶し、同じ順番で再現。後半は4秒の遅延挿入 | Corsiブロックタッピング（Corsi, 1972） | 視覚性短期記憶、空間知覚、プランニング |

**Manas設計への示唆：** WOM-ASM（数列記憶）は言語依存度が高く、テキスト理解が困難な重度ID児には不適。Manasでは**視覚的シーケンス記憶**（色や形の順番を覚える）に置き換えるべき。VISMEM-PLAN（Corsiブロック相当）は**非言語・視覚空間ベースで、ID/ASD児にそのまま適用可能な最有力候補**。遅延条件の有無で難易度を階層化できる設計も優れている。

### 1-2. 実行機能・注意系タスク

| タスク名（コード） | 内容 | 元になった検査 | 測定スキル |
|---|---|---|---|
| **迷路テスト（VIPER-PLAN）** | 迷路を最少ステップ・最短時間でゴールまでたどる | Porteus迷路（Porteus, 1950）+ NEPSY Route Finding | プランニング、空間知覚、視覚走査 |
| **ストループテスト（INH-REST）** | 色と文字が不一致の刺激に対し正しく応答（例：赤で書かれた「青」→「赤」を選択） | 古典的ストループ検査 | 抑制、処理速度、シフティング |
| **同時処理テスト（DIAT-SHIF）** | ボールの動きを追従しながら、同時に色変化に反応する二重課題 | Vienna Test System（Whiteside, 2002）ベース | 分割的注意、シフティング、手と目の協応 |
| **処理テスト（REST-INH）** | 数字と図形の組み合わせで、ルールに基づき応答を抑制する | Go/No-Go課題の変形 | 抑制、処理速度 |
| **TMTテスト** | 数字→文字を交互に結ぶ（Trail Making Test） | TMT（Reitan, 1955） | 処理速度、シフティング、視覚走査 |

**Manas設計への示唆：** VIPER-PLAN（迷路）は**最も汎用的で年齢・能力レベルを問わず適用可能**。迷路のサイズ（2×2→10×10）と行き止まりの数で、極端に低い認知レベルから高い水準まで難易度をシームレスに調整できる。DIAT-SHIF（二重課題）はID児には認知負荷が高すぎる可能性があるが、ADHD児には適切。Manasでは**単一課題モード→二重課題モードへの段階的移行**を組み込むべき。ストループテスト（INH-REST）は文字の読みが前提のため、**非言語版（色×形の不一致）に変換する必要がある**。

### 1-3. 知覚・協調系タスク

| タスク名（コード） | 内容 | 元になった検査 | 測定スキル |
|---|---|---|---|
| **推定テスト（EST-I）** | 動く物体の速度から到達位置・時間を予測 | 独自設計 | 推定、処理速度 |
| **推定テスト（EST-II）** | 聴覚刺激の長さを再現（音の持続時間を推定） | Duration Pattern Test（Frota & Pereira, 2003） | 推定、聴覚知覚 |
| **推定テスト（EST-III）** | 視覚的な距離・奥行きの推定 | 独自設計 | 推定、空間知覚 |
| **協応テスト（HECOOR）** | 画面上のボールをタップ/クリックで追従 | 独自設計 | 手と目の協応、反応時間 |
| **同期テスト（UPDA-SHIF）** | 経路上を移動するボールを正確にトラッキング | Vienna Test System（VST） | 手と目の協応、更新 |
| **視野テスト（WIFIVI）** | 中央注視点を見ながら周辺視野の刺激を検出 | UFOV（Useful Field of View） | 視野の幅、知覚 |
| **デコーディングテスト（VIPER-NAM）** | 表示された物体の名前の頭文字を4つの選択肢から選ぶ | Boston Naming Test + WAIS-III語彙 | 視覚知覚、命名、処理速度 |
| **識別テスト（COM-NAM）** | 以前に見た/聞いたオブジェクトかどうかを判定 | Boston Naming Test（Kaplan et al., 1983） | 再認、命名 |
| **質問テスト（REST-COM）** | 画像と単語の対応を判断 | 独自設計 | 分割的注意、処理速度 |

**Manas設計への示唆：** HECOOR（ボール追従）とUPDA-SHIF（トラッキング）は**微細運動の評価・訓練として優秀で、言語完全非依存**。感覚過敏のあるASD児には、ボールの動きの速度・色・軌道の予測可能性を調整する追加パラメータが必要。VIPER-NAM（頭文字選択）は知的障害児にはハードルが高い。Manasでは**画像マッチング（同じ物を選ぶ）→カテゴリマッチング（仲間を選ぶ）→頭文字マッチング**の3段階に分解すべき。

## Part 3：CogniFitのゲーム設計原則の抽出

### 3-1. タスク-スキル マッピングの構造

全ゲームのマッピングを集計すると、以下のパターンが見える：

**最頻出スキル（多くのゲームで訓練される）：**
1. 空間知覚（Spatial Perception）— 約25ゲーム以上
2. プランニング（Planning）— 約20ゲーム以上
3. 反応時間（Response Time）— 約15ゲーム以上
4. 手と目の協応（Hand-eye Coordination）— 約12ゲーム
5. 視覚知覚（Visual Perception）— 約12ゲーム
6. 推定（Estimation）— 約12ゲーム

**低頻度スキル（少数のゲームでしか訓練されない）：**
1. 聴覚知覚 — 4〜5ゲーム
2. 命名 — 5〜6ゲーム
3. 文脈記憶 — 2〜3ゲーム
4. 視野の幅 — 1〜2ゲーム

→ **CogniFitは空間知覚・プランニング・反応時間に過度に依存しており、記憶の下位分類や聴覚系の訓練は手薄。** Manasでは、ワーキングメモリと感覚処理（特に聴覚・触覚）の訓練ゲームを厚くする差別化機会がある。

### 3-2. 難易度調整の変数（ゲーム横断的パターン）

| 難易度パラメータ | 適用ゲーム例 | 効果 |
|---|---|---|
| **アイテム数** | Water Lilies, Candy Factory, WOM-ASM | 記憶負荷の段階的増加 |
| **速度** | Reaction Field, Color Rush, 全テニス系 | 処理速度・反応時間の要求変更 |
| **遅延時間** | VISMEM-PLAN | 記憶保持時間の延長 |
| **妨害刺激の比率** | Happy Hopper, Reaction Field | 抑制要求の増加 |
| **同時処理数** | Traffic Manager, Crossroads, DIAT-SHIF | 分割注意の負荷増加 |
| **パターンの複雑さ** | IQbe, Synaptix, Candy Line Up | 推論・パターン認識の難度 |
| **迷路サイズ** | Penguin Explorer, VIPER-PLAN | プランニングの問題空間拡大 |
| **精度要求** | Neon Lights, Mouse Challenge, HECOOR | 微細運動の精度要求 |

**Manas設計への示唆：** CogniFitは「正答率70〜80%を維持」するゾーンに難易度を調整するが、ID児研究（前回分析のPMC4542133）は「より高い正答率（90%+）での維持が有効」と示唆している。Manasの適応アルゴリズムは**成功体験率を80〜90%に設定し、エラー時の即時フィードバック＋視覚的手がかり提示**を組み込むべき。

### 3-3. セッション構造の設計原則

CogniFitのセッション構造は以下の通り：

```
[トレーニングゲーム1] → [トレーニングゲーム2] → [評価タスク1]
         ↕                    ↕                    ↕
      約5〜7分             約5〜7分             約3〜5分
```

- 合計15〜20分（子ども向けは10〜15分）
- 週3回、非連続日
- AIが次セッションのゲーム選択と難易度を決定

**Manas設計への示唆：** 重度ID児には10〜15分でも長い可能性がある。**5分×2セッション/日**のマイクロセッション構造を検討すべき。また、CogniFitの「毎セッション評価タスク1つ」の設計は、継続的モニタリングに不可欠だが、ID児には評価タスクの負荷を下げる（例：2セッションに1回、またはゲーム内データから暗黙的に評価）工夫が要る。

## Part 6：まとめ — CogniFitから学ぶべき10の設計原則

1. **各ゲームは2〜4の認知スキルに明示的にマッピングする** — 「何を鍛えているか」の透明性がエビデンスと専門家の信頼を得る鍵
2. **評価タスクは古典的な神経心理学的検査に基づいて設計する** — Manasの評価タスクもCorsiブロック、Go/No-Go、迷路課題など確立されたパラダイムを出発点にする
3. **多対多マッピングで効率的にスキルをカバーする** — 15ゲームで20スキルをカバーすることも可能
4. **難易度調整は「アイテム数」「速度」「遅延」「妨害刺激比」「精度要求」の5変数で制御する** — これらの組み合わせで無限の難易度グラデーションを生成
5. **セッションに評価タスクを組み込む** — トレーニングと評価を統合し、継続的モニタリングを実現
6. **同一エンジン×異なるパラメータで複数ゲームを生成する** — テニス系3種のように、開発効率を最大化
7. **言語依存度を意識的にコントロールする** — CogniFitの60+ゲームのうち約50は言語依存度「低」or「なし」。Manasでは「なし」をデフォルトにすべき
8. **各ゲームに明示的な神経心理学的根拠を記載する** — 保護者・専門家への説明資料として不可欠
9. **ゲーミフィケーションをCogniFitより強化する** — CogniFitの弱点。Duolingoライクなストリーク・バッジ・XPシステムをManasの差別化要素にする
10. **週3回×15〜20分のセッション構造を基本とし、対象群に応じて短縮する** — ID児向けには5〜10分×毎日のマイクロセッションを検討
