# 修正指示書: ゲーム記録保存 & プレイエリア配置

## 修正1: ゲーム記録がSupabaseに保存されない（最優先）

### 症状
- **混合セッション**（「あそぶ」ボタン→ `/session/play`）でプレイしても「きろく」タブに反映されない
- **フリープレイ**（個別ゲーム→ `/game/[integratedId]`）でも同様に反映されない
- **両方で保存されない**ため、共通の記録保存ロジック自体に問題がある可能性が高い

### 調査手順

#### Step 1: 共通の記録保存関数を特定
- `useGameSession` フック（もしくはそれに相当する記録保存の共通関数）を確認
- ゲーム終了時に呼ばれる保存処理のエントリーポイントを追跡

#### Step 2: childId の取得タイミングを確認
- 前回の修正で「childIdの非同期問題」が指摘されている
- `childId` が `null` や `undefined` のまま保存関数が呼ばれていないか確認
- Supabaseからの取得が完了する前にゲーム開始→終了→保存のフローが走っている可能性

#### Step 3: Supabase書き込みの実行確認
- 保存関数内で実際に `supabase.from('テーブル名').insert()` もしくは `.upsert()` が呼ばれているか
- 呼ばれている場合、エラーハンドリングでエラーが握りつぶされていないか
- `console.error` や `console.log` を追加して、保存時の `data` と `error` を両方出力するようにする

#### Step 4: 「きろく」タブの読み取り側も確認
- 書き込みが成功していても、読み取りクエリが別テーブルや別カラムを見ている可能性
- 保存先テーブル/カラムと、きろくタブの読み取りクエリが一致しているか確認

### 修正方針
1. `childId` が確実に取得された後でのみ保存処理を実行するガードを入れる
2. 保存処理に明示的なエラーログを追加
3. 混合セッション（`session-engine.ts` / `mixed-session-manager.ts`）のゲーム完了コールバック内で記録保存関数が呼ばれているか確認。呼ばれていなければ追加
4. フリープレイ（`/game/[integratedId]/page.tsx`）のゲーム完了コールバックも同様に確認
5. 修正後、ブラウザのDevToolsコンソールで保存成功ログが出ることを確認

### 検証方法
1. `npm run build` & `tsc --noEmit` でビルドエラーなし
2. フリープレイで1ゲームプレイ → 「きろく」タブに反映されること
3. 混合セッションで1セッション完了 → 「きろく」タブに反映されること
4. ブラウザDevToolsでSupabaseへのPOSTリクエストが飛んでいること

## 修正2: プレイエリアの配置を画面中央寄りにする

### 症状
- プレイエリアが画面上部に偏って配置されている
- 操作しづらい位置にある

### 修正方針
- プレイエリアのコンテナを **垂直方向で画面中央寄り** に配置する
- 正方形の制約は不要。**少し縦長のアスペクト比でもOK**
- ナビバーやヘッダーを考慮した上で、残りのスペースの中央に配置

### 具体的な実装
- プレイエリアを包むコンテナに以下のようなスタイルを適用:
  ```css
  /* 例: flexで垂直中央寄せ */
  .game-container {
    display: flex;
    flex-direction: column;
    justify-content: center;  /* 垂直中央 */
    align-items: center;
    min-height: calc(100vh - ナビバー高さ - ヘッダー高さ);
    /* もしくは min-height: calc(100dvh - ...) でモバイル対応 */
  }
  ```
- プレイエリア自体のサイズ制約:
  - `max-width: 100%` は維持
  - `aspect-ratio: 1/1` の制約があれば **削除** するか `aspect-ratio: 3/4` 程度に変更
  - 縦長を許容するが、横幅は画面幅いっぱいを維持

### 対象ファイル
- 各ゲームの共通レイアウトコンポーネント（おそらく `GameLayout` や各ゲームの `page.tsx` 内のコンテナ）
- 混合セッションのプレイ画面: `src/app/session/play/page.tsx`
- フリープレイのプレイ画面: `src/app/(game)/game/[integratedId]/page.tsx`

### 検証方法
1. `npm run build` & `tsc --noEmit` でビルドエラーなし
2. スマホ実機（もしくはDevToolsモバイルエミュレータ）で全ゲームを起動し、プレイエリアが画面中央付近に表示されること
3. ボトムナビと被らないこと
4. 縦長の端末でも横長の端末でも破綻しないこと

---

## 共通注意事項
- 修正後は必ず `npm run build` と `tsc --noEmit` を通すこと
- 既存のゲームロジックを壊さないよう、変更範囲を最小限に
- デプロイ: `git add -A && git commit -m "fix: game records saving & play area centering" && git push origin main`
